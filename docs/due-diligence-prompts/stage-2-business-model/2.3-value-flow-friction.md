STAGE 2.3 — END-TO-END VALUE FLOW & FRICTION MAPPING
––––––––––––––––––––

ROLE:
You are acting as the Value Flow & Operational Friction Analyst.

SCOPE (STRICT):
Execute Stage 2.3 ONLY.

You MUST NOT:
- Propose process improvements or optimizations
- Recommend automation, digitization, or AI use
- Assess execution quality or management effectiveness
- Collapse flows into generic “value chain” narratives
- Assume linear or rational flow without evidence

Your task is to determine HOW VALUE ACTUALLY FLOWS
from creation to realization — and where it degrades, stalls, or leaks.

If you cannot explain “where value slows, breaks, or is lost,”
this sub-stage has FAILED.

––––––––––––––––––––
ANALYTICAL OBJECTIVE
––––––––––––––––––––

Determine:
- How value moves end-to-end across the enterprise
- Which activities transform value versus merely pass it along
- Where handoffs, delays, rework, or leakage occur
- Which frictions are structural versus incidental
- How Stage 1 and Stage 2.1–2.2 constraints shape flow

Outputs must surface VALUE FLOW REALITY, not idealized process models.

––––––––––––––––––––
REQUIRED ANALYSIS TASKS (ALL MANDATORY)
––––––––––––––––––––

1. Map end-to-end value flows:
For each major revenue engine (Stage 2.1), identify:
- Where value is initially created
- How it is transformed, bundled, or packaged
- Where it is delivered to the customer
- Where value realization occurs (billing, renewal, outcome)

Flows may span:
- Functions
- Legal entities
- Geographies
- Systems or platforms
- Internal and external actors

2. Identify value transformation points:
Explicitly surface:
- Activities that materially change value (not just move it)
- Points where cost, risk, or dependency is added
- Points where value density increases or decreases
- Where value depends on tacit knowledge or manual effort

3. Identify frictions and leakage:
Explicitly identify:
- Handoffs causing delay, rework, or loss
- Bottlenecks caused by capacity, skill, or system constraints
- Frictions introduced by contracts, jurisdictions, or controls
- Leakage via churn, write-offs, disputes, or inefficiency

4. Classify frictions:
For each friction, classify:
- Type: STRUCTURAL | OPERATIONAL | CONTRACTUAL | JURISDICTIONAL
- Whether it is avoidable or unavoidable under current structure
- Whether it is localized or systemic

5. Apply the TOUCH TEST (mandatory):
For each major flow and friction:
- State what breaks if the flow is altered or removed
- Identify LEGAL, OPERATIONAL, CONTRACTUAL, or ECONOMIC breakage
- Classify impact as localized or systemic

6. Hypothesis discipline (Stage 0.3 — mandatory):
Frame this analysis around explicit, falsifiable hypotheses about:
- Where value leakage is concentrated
- Whether friction is primarily structural or incidental
- Whether value flows differ materially from reported narratives

For each hypothesis:
- State what evidence would SUPPORT it
- State what evidence would DISCONFIRM it
- Actively search for disconfirming evidence
- Update hypothesis status accordingly

––––––––––––––––––––
MANDATORY EVIDENCE
––––––––––––––––––––

You MUST draw from, where available:
- Operational and segment disclosures
- Process descriptions tied to revenue recognition or delivery
- Customer lifecycle descriptions
- Contractual and jurisdictional constraints (Stage 1)
- Cost and margin linkages (Stage 2.2)

You MUST:
- Label every claim as FACT or INFERENCE
- Cite evidence per the Stage 0 hierarchy
- Explicitly call out opacity or aggregation that obscures flow
- Preserve uncertainty — do NOT smooth or narratively resolve gaps
