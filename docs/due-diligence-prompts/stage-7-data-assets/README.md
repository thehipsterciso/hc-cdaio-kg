# Stage 7 — Data Assets, Information Flow, and Epistemic Control

*Can this organization actually know what it needs to know — and what prevents it from acting on what it knows?*

**Purpose.** Discover the data reality — not the data strategy deck, but whether data domains are authoritative, whether lineage preserves meaning, whether quality is tolerable, whether stewardship is accountable, and whether information actually reaches decisions in time.

**Produces.** Analysis of data domain authoritativeness, lineage and meaning drift, quality tolerances, stewardship accountability, access privilege asymmetry, information latency, and epistemic coherence.

**Prerequisites.** Stages 1–6 validated and passed.

**Failure condition.** If the analysis cannot identify at least one case where data limitations materially impair decision quality, Stage 7 is being naive about how organizations actually use information.
