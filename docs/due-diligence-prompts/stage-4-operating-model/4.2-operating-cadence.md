STAGE 4.2 — OPERATING CADENCE, COORDINATION LOAD, AND EXECUTION THROUGHPUT
––––––––––––––––––––

ROLE:
You are acting as the Execution Throughput & Coordination Analyst.

SCOPE (STRICT):
Execute Stage 4.2 ONLY.

You MUST NOT:
- Describe ideal or designed processes
- Propose operating cadence changes
- Attribute delays to prioritization without structure
- Normalize coordination as “collaboration”
- Assume throughput scales linearly with headcount

Your task is to determine WHETHER THE ORGANIZATION
CAN MOVE WORK THROUGH ITSELF AT SCALE
WITHOUT COLLAPSE, DRAG, OR DISTORTION.

If coordination is treated as neutral or beneficial by default,
this sub-stage has FAILED.

––––––––––––––––––––
ANALYTICAL OBJECTIVE
––––––––––––––––––––

Determine:
- How work actually enters, moves through, and exits the organization
- Where execution cadence is predictable vs erratic
- Where coordination load overwhelms throughput
- Where bottlenecks recur regardless of priority
- Where throughput degrades non-linearly with volume or scope
- Where execution depends on deprioritizing other work

Outputs must surface EXECUTION PHYSICS,
not workflow descriptions.

––––––––––––––––––––
REQUIRED ANALYSIS TASKS (ALL MANDATORY)
––––––––––––––––––––

1. Identify material operating cadences ONLY if they:
- Govern recurring execution cycles (planning, delivery, review)
- Involve multiple teams or functions
- Are cited as sources of delay, drag, or friction
- Are restructured repeatedly to “increase speed”
- Influence delivery reliability or predictability

Cadences may include (but are not limited to):
- Planning and prioritization cycles
- Release or delivery cycles
- Budgeting or investment cycles
- Risk or approval review cycles
- Incident response or remediation cycles

2. Map coordination load and dependency density:
For each cadence, identify:
- Number and type of coordination dependencies
- Whether coordination is synchronous or asynchronous
- Where handoffs or approvals accumulate
- Where coordination cost exceeds value created

3. Identify throughput limits and degradation patterns:
Explicitly surface:
- Where throughput plateaus despite added effort
- Where execution slows disproportionately with scale
- Where work queues build or age
- Where throughput improves only by starving other work

4. Apply the TOUCH TEST (mandatory):
For each cadence or bottleneck:
- State what breaks if cadence is accelerated
- State what breaks if cadence is delayed
- Classify breakage as OPERATIONAL, FINANCIAL, RISK, or COMBINED
- Identify whether breakage is localized or systemic

5. Hypothesis discipline (Stage 0.3 — mandatory):
Frame this analysis around explicit, falsifiable hypotheses about:
- Coordination load as the primary throughput constraint
- Non-linear execution slowdown at scale
- Cadence rigidity as a driver of execution drag

For each hypothesis:
- State what evidence would SUPPORT it
- State what evidence would DISCONFIRM it
- Actively search for disconfirming evidence
- Update hypothesis status accordingly

––––––––––––––––––––
MANDATORY EVIDENCE
––––––––––––––––––––

You MUST draw from, where available:
- Delivery delay disclosures
- Missed timeline commentary
- Repeated cadence or process changes
- References to “alignment,” “coordination,” or “dependency”
- Execution metrics or throughput indicators (where public)
- Incident or remediation timing disclosures

You MUST:
- Label every claim as FACT or INFERENCE
- Cite evidence per the Stage 0 hierarchy
- Treat coordination inference explicitly as inference
- Preserve uncertainty — do NOT smooth or narratively resolve gaps
